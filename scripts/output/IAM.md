

# IAM Security Controls Matrix

| IAM-006 | Role-Based Access Control Implementation | Design and implement role-based access control models with formally defined roles aligned to job functions, separation of duties requirements, and organizational structure across all platforms | Preventive | AC-2, AC-3, AC-6(7) | A.5.15, A.5.18 | 5.4, 6.8 | IAM-08 | DSS05.04, DSS06.03 | ISM-1503, ISM-1507 | Roles serve as policy enforcement constructs; role assignments continuously validated against current context | RBAC models normalized across cloud provider native IAM systems and on-premises directories with centralized role catalog | IAM Team / HR |
| IAM-007 | Attribute-Based Access Control | Implement attribute-based access control policies that incorporate contextual attributes such as user risk score, device posture, location, time, data sensitivity, and threat intelligence for dynamic authorization decisions | Preventive | AC-3(13), AC-3(14) | A.5.15 | 6.7, 6.8 | IAM-09 | DSS05.04 | ISM-0988 | Core Zero Trust policy engine capability; every access decision informed by real-time contextual attributes and risk signals | ABAC policies portable across cloud environments using standardized policy languages (XACML, Cedar, OPA/Rego) with consistent attribute sources | IAM Team / Security Architecture |
| IAM-008 | Identity Lifecycle Management | Automate the full identity lifecycle including provisioning, modification, suspension, and deprovisioning of all identity types, triggered by authoritative HR or business events with defined SLAs | Preventive | AC-2, PS-4, PS-5 | A.5.16, A.5.18, A.6.5 | 5.1, 5.2, 5.3 | IAM-02, IAM-03 | DSS05.04, DSS06.03 | ISM-0430, ISM-0407 | Stale or orphaned identities represent implicit trust violations; automated lifecycle ensures only active verified identities exist | Lifecycle management orchestrated across all cloud tenants, SaaS applications, and on-premises systems through SCIM, APIs, and identity governance platform | IAM Team / HR |
| IAM-009 | Timely Access Revocation | Ensure immediate or near-real-time revocation of access upon role change, termination, contract completion, or security incident, including revocation of sessions, tokens, and cached credentials | Preventive / Corrective | AC-2(2), PS-4, PS-5 | A.5.18, A.6.5 | 5.2, 5.3 | IAM-03 | DSS05.04 | ISM-0430, ISM-0407 | Zero Trust requires continuous authorization; revocation must propagate instantly to all relying parties and enforcement points | Revocation propagated across all cloud identity providers, federated trust relationships, SaaS applications, and on-premises directories simultaneously | IAM Team / HR / Security Operations |
| IAM-010 | Access Certification and Recertification | Conduct periodic access reviews and recertification campaigns for all user, service, and privileged accounts with risk-based frequency, manager and data owner attestation, and automated remediation of non-certified access | Detective / Corrective | AC-2(13), AC-6(7) | A.5.15, A.5.18 | 5.1, 5.4 | IAM-05 | MEA01.03, DSS05.04 | ISM-1503 | Continuous verification of access entitlements; trust is time-bounded and must be periodically revalidated | Access certification spans all cloud environments, SaaS applications, and on-premises systems with aggregated entitlement views | IAM Governance / Application Owners |
| IAM-011 | Service Account and Non-Human Identity Management | Manage all service accounts, API keys, managed identities, service principals, and machine credentials with dedicated lifecycle controls, ownership assignment, credential rotation, and privilege restrictions | Preventive | AC-2(1), IA-4, IA-5 | A.5.16, A.5.18 | 5.4, 6.1 | IAM-10 | DSS05.04 | ISM-1685, ISM-1686 | Non-human identities must be uniquely identified, authenticated, and authorized with same rigor as human identities; no implicit trust | Service account management normalized across cloud-native service principals (AWS roles, Azure managed identities, GCP service accounts) and on-premises service accounts | IAM Team / Application Owners |
| IAM-012 | Credential Management and Protection | Establish comprehensive credential management policies covering password complexity, rotation, storage, transmission, and protection against credential stuffing, spraying, and replay attacks | Preventive | IA-5, IA-5(1), IA-5(2) | A.5.17, A.8.5 | 5.2, 6.3 | IAM-06 | DSS05.04 | ISM-0421, ISM-0422 | Credentials alone are insufficient for trust; combined with continuous verification signals | Credential policies enforced uniformly across all cloud and on-premises identity stores; secrets centrally managed in cloud-agnostic vaults | IAM Team / Security Operations |
| IAM-013 | Secrets Management | Implement centralized secrets management for storing, distributing, rotating, and auditing application secrets, API keys, certificates, encryption keys, and database credentials with automated rotation capabilities | Preventive | IA-5(7), SC-12, SC-28 | A.5.17, A.8.24 | 5.2, 6.1 | IAM-10, EKM-01 | DSS05.04 | ISM-0457, ISM-1685 | Eliminate hardcoded secrets; dynamically issued short-lived credentials aligned with Zero Trust principles | Secrets management solution integrated across all cloud environments (AWS Secrets Manager, Azure Key Vault, GCP Secret Manager, HashiCorp Vault) with unified policy | DevOps / Security Engineering |
| IAM-014 | Separation of Duties | Enforce separation of duties through technical controls preventing any single individual from performing conflicting critical functions such as request-approve, develop-deploy, or administer-audit | Preventive | AC-5, AC-6(2) | A.5.3, A.5.15 | 5.4, 6.8 | IAM-05, IAM-08 | DSS06.03, MEA02.01 | Conflict detection and enforcement as part of continuous policy evaluation; no implicit trust in any single actor | SoD rules enforced across cloud IAM platforms, ERP systems, and on-premises infrastructure with centralized conflict detection | IAM Governance / Internal Audit |
| IAM-015 | Emergency and Break-Glass Access | Define and implement emergency access (break-glass) procedures with pre-provisioned accounts secured by strong controls, multi-party authorization, comprehensive logging, and mandatory post-incident review and revocation | Corrective | AC-2(2), CP-2 | A.5.18 | 5.4 | IAM-04 | DSS05.04 | ISM-1508 | Break-glass accounts continuously monitored; activation triggers immediate alerting and forensic logging; trust is temporary and audited | Break-glass procedures defined for each cloud environment and on-premises infrastructure with cross-platform alerting and unified audit trail | Security Operations / IAM Team |
| IAM-016 | Single Sign-On Implementation | Deploy enterprise single sign-on using federated identity standards (SAML 2.0, OIDC, OAuth 2.0) to consolidate authentication, reduce credential sprawl, and enable centralized policy enforcement | Preventive | IA-2, IA-8 | A.5.16, A.8.5 | 6.3, 12.5 | IAM-01, IAM-06 | DSS05.04 | ISM-1546 | Centralized authentication enables consistent policy enforcement, risk-based access decisions, and continuous session evaluation | SSO federation extended to all cloud providers, SaaS applications, and on-premises applications through centralized identity provider with protocol bridging | IAM Team / Enterprise Architecture |
| IAM-017 | Adaptive and Risk-Based Authentication | Implement adaptive authentication mechanisms that dynamically adjust authentication strength and step-up requirements based on real-time risk scoring incorporating user behavior, device posture, network context, and threat intelligence | Preventive / Detective | IA-2(13), IA-10, SI-4 | A.8.5 | 6.3, 6.5 | IAM-06 | DSS05.04 | ISM-1401 | Core Zero Trust capability; authentication strength must match assessed risk; continuous re-evaluation during session | Adaptive authentication policies applied consistently across all cloud platforms, SaaS, and on-premises access through centralized risk engine | IAM Team / Security Architecture |
| IAM-018 | Identity Federation and Trust Management | Establish and manage identity federation trust relationships with external partners, customers, and cloud providers using formal trust agreements, certificate management, and continuous monitoring of federated assertions | Preventive | IA-8, IA-8(1), IA-8(4) | A.5.19, A.5.20 | 6.3, 16.1 | IAM-01 | DSS05.04, APO09.03 | ISM-1546 | Federated trust boundaries explicitly defined and continuously validated; no implicit trust in external identity assertions | Federation standards (SAML, OIDC) used to establish trust across cloud providers and external partners with centralized trust management | IAM Team / Third-Party Risk |
| IAM-019 | Directory Services Security | Secure all directory services (Active Directory, LDAP, cloud directories) with hardened configurations, replication security, schema protection, administrative tier isolation, and continuous monitoring for directory-based attacks | Preventive / Detective | AC-3, SC-4, SI-4 | A.8.9, A.8.12 | 4.1, 5.4 | IAM-01, IAM-04 | DSS05.04, DSS05.05 | ISM-1055, ISM-1620 | Directory services are critical trust anchors; must be hardened, segmented, and continuously monitored | Directory services architecture spans on-premises AD, Azure AD/Entra ID, AWS Directory Service, and GCP Cloud Identity with secure synchronization and tiered administration | Infrastructure / IAM Team |

| IAM-021 | Identity Threat Detection and Response | Implement identity-specific threat detection capabilities monitoring for credential compromise, identity-based attack patterns (Kerberoasting, pass-the-hash, token theft, OAuth abuse), anomalous authentication, and automated response playbooks | Detective / Corrective | SI-4, IR-4, IR-5 | A.8.15, A.8.16 | 8.5, 8.11 | IAM-13 | DSS05.07, DSS02.01 | ISM-0120, ISM-1228 | Assume breach posture; continuously detect and respond to identity-based attacks in real time | Identity threat detection aggregates signals from all cloud providers, on-premises directories, and SaaS platforms into unified ITDR platform | Security Operations / IAM Team |

| IAM-023 | Token and Session Management | Implement secure token and session management including short-lived tokens, token binding, refresh token rotation, session timeout enforcement
