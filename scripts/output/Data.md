

# Data Security Controls Matrix

| DATA-007 | Data Loss Prevention (DLP) | Deploy DLP controls at network, endpoint, email, cloud, and application layers to detect, alert, and block unauthorized exfiltration of classified data based on content, context, and user behavior | Detective / Preventive | SC-7(10), SI-4, AC-4 | A.8.12 | 3.13 | DSP-08 | DSS05.02, DSS05.07 | ISM-0947 | Continuous inspection of data flows regardless of source/destination; no trusted channel exemptions | DLP policies must be unified across SaaS, IaaS, endpoints, and on-premises with consistent rule sets and centralized management | Security Operations Lead |
| DATA-008 | Data Masking and Tokenization | Apply dynamic or static data masking, tokenization, or pseudonymization to sensitive data elements in non-production environments, analytics workloads, and external-facing interfaces | Preventive | SC-28, SI-19 | A.8.11 | 3.12 | DSP-05 | APO14.08, DSS05.03 | ISM-0457 | Minimizes sensitive data exposure aligned with least-privilege data access; reduces blast radius | Masking and tokenization solutions must operate consistently across cloud databases, data warehouses, and on-premises systems | Data Governance Officer |
| DATA-009 | Data Access Control and Authorization | Enforce role-based, attribute-based, or policy-based access controls on all data repositories ensuring least privilege, separation of duties, and just-in-time access provisioning | Preventive | AC-3, AC-6, AC-2 | A.5.15, A.8.3 | 3.3, 6.8 | DSP-10, IAM-02 | DSS05.04, APO14.06 | ISM-0432, ISM-1175 | Per-request authorization decisions based on identity, device health, context, and data sensitivity | Access control policies must be enforceable via cloud IAM, database-native controls, and identity-aware proxies across environments | Identity and Access Management Lead |
| DATA-010 | Data Integrity Verification | Implement cryptographic hashing, checksums, digital signatures, and integrity monitoring to detect unauthorized modification of critical data at rest and in transit | Detective | SI-7, SC-8(1) | A.8.24 | 3.14 | DSP-17 | DSS05.03, DSS06.02 | ISM-1187 | Continuous integrity verification ensures data has not been tampered with; no implicit trust in data provenance | Integrity verification mechanisms must span cloud storage, databases, file systems, and cross-cloud replication pipelines | Security Architecture Lead |
| DATA-011 | Database Activity Monitoring | Deploy real-time database activity monitoring (DAM) for all critical databases capturing privileged operations, schema changes, bulk data access, anomalous queries, and policy violations | Detective | AU-12, SI-4 | A.8.15, A.8.16 | 3.14, 8.5 | DSP-08 | DSS05.07, MEA01.03 | ISM-0585 | Continuous monitoring of data access patterns detects lateral movement and privilege abuse in Zero Trust architectures | DAM solutions must support cloud-managed databases (RDS, Cloud SQL, Cosmos DB), on-premises databases, and data warehouses | Security Operations Lead |
| DATA-012 | Data Backup and Recovery | Implement automated, encrypted, and geographically redundant backup processes with defined RPO/RTO targets, regular restoration testing, and immutable backup copies to protect against ransomware | Corrective | CP-9, CP-10 | A.8.13 | 11.2, 11.4 | DSP-14, BCR-11 | DSS04.07, DSS04.08 | ISM-1511, Essential 8: Regular Backups | Immutable backups ensure recovery capability even when primary environments are fully compromised | Backup strategies must account for multi-cloud data residency, cross-region replication, and cloud-native backup services | Infrastructure Operations Lead |
| DATA-013 | Data Retention and Disposal | Enforce data retention policies with automated lifecycle management ensuring data is retained only for required periods and securely destroyed using approved sanitization methods upon expiry | Preventive | SI-12, MP-6, SR-12 | A.8.10 | 3.4 | DSP-15, DSP-16 | APO14.09, BAI09.03 | ISM-0348, ISM-0363 | Minimizes attack surface by reducing unnecessary data persistence; aligns with data minimization principles | Retention and disposal automation must function across cloud object stores, managed databases, SaaS platforms, and on-premises storage | Data Governance Officer |
| DATA-014 | Data Sovereignty and Residency Controls | Implement technical controls to enforce data residency requirements based on jurisdictional, regulatory, and contractual obligations, preventing unauthorized data transfer across boundaries | Preventive | AC-4, SA-9(5) | A.5.34, A.5.35 | 3.1 | DSP-19 | APO14.05, MEA03.03 | ISM-0861 | Policy enforcement at data layer ensures compliance regardless of network topology or deployment location | Cloud resource placement policies, geo-fencing, data boundary services, and replication controls must be enforced per provider | Data Governance Officer |
| DATA-015 | Sensitive Data Discovery and Scanning | Deploy automated scanning tools to continuously discover, identify, and classify sensitive data (PII, PHI, PCI, credentials, secrets) across all repositories, file shares, and cloud storage | Detective | RA-5, PM-5 | A.5.12, A.8.10 | 3.1, 3.7 | DSP-01, DSP-02 | APO14.01, MEA01.03 | ISM-0661 | Continuous discovery ensures no shadow data escapes Zero Trust policy enforcement | Scanning must cover cloud object storage, databases, SaaS applications, code repositories, and on-premises file systems | Data Governance Officer |
| DATA-016 | Privacy and Personal Data Protection | Implement technical and organizational measures for personal data protection including consent management, purpose limitation, data subject access request (DSAR) workflows, and privacy impact assessments | Preventive | PT-2, PT-3, PT-4 | A.5.34 | 3.1 | DSP-03, DSP-04 | APO14.05, MEA03.04 | ISM-0839 | Privacy controls ensure data is accessed only for legitimate, verified purposes aligned with least-privilege principles | Privacy controls must integrate with multi-cloud data processing activities and SaaS application data handling | Data Privacy Officer |
| DATA-017 | Secure Data Sharing and Transfer | Enforce approved secure methods for data sharing including encrypted file transfer, secure APIs, managed file transfer (MFT), and data clean rooms with audit trails and access revocation | Preventive | AC-4, AC-21, SC-8 | A.5.14, A.8.24 | 3.10, 3.12 | DSP-06 | DSS05.02, DSS06.06 | ISM-0947, ISM-0489 | All data transfers are authenticated, authorized, encrypted, and logged regardless of trust boundary | Secure sharing mechanisms must function across cloud-to-cloud, cloud-to-on-premises, and B2B scenarios with consistent policy | Data Governance Officer |
| DATA-018 | Data Labeling and Tagging Enforcement | Implement automated and manual data labeling mechanisms that apply metadata tags for classification, sensitivity, retention, and regulatory scope with enforcement at creation and modification | Preventive | SC-16, RA-2 | A.5.12, A.5.13 | 3.7 | DSP-01 | APO14.01, APO14.04 | ISM-0661 | Labels drive automated policy enforcement for access, encryption, DLP, and sharing in Zero Trust models | Labeling must integrate with cloud-native tagging systems, document management platforms, and email gateways across all environments | Data Governance Officer |
| DATA-019 | Privileged Data Access Management | Implement just-in-time, time-bound, and approval-based privileged access to sensitive data stores with session recording, enhanced monitoring, and automatic credential rotation | Preventive | AC-2(2), AC-6(1), AC-6(2) | A.8.2, A.8.18 | 6.5, 6.8 | DSP-10, IAM-04 | DSS05.04 | ISM-1175, ISM-1507 | Never-trust privileged users; verify continuously with enhanced controls, session monitoring, and ephemeral credentials | Privileged access workflows must integrate with cloud IAM, PAM solutions, and database-native privilege controls across platforms | Identity and Access Management Lead |
| DATA-020 | Data Anonymization and De-identification | Apply formally validated anonymization or de-identification techniques (k-anonymity, l-diversity, differential privacy) before sharing data for analytics, AI/ML training, or third-party use | Preventive | SI-19, PT-6 | A.8.11 | 3.12 | DSP-05 | APO14.08 | ISM-0457 | Reduces risk of re-identification and ensures data utility without exposing sensitive attributes; supports least-exposure principle | Anonymization must be applied consistently across cloud analytics platforms, data lakes, and AI/ML training pipelines in all environments | Data Privacy Officer |
| DATA-021 | Secure Database Configuration | Enforce hardened database configurations including default credential removal, unnecessary feature/service disablement, parameterized queries, audit logging, and network access restriction | Preventive | CM-6, CM-7, SC-3 | A.8.9 | 4.8, 16.11 | DSP-09 | DSS05.03, BAI10.01 | ISM-1247 | Every database instance is treated as untrusted; hardened regardless of network position per Zero Trust principles | Configuration baselines must be defined and enforced for cloud-managed databases, containerized databases, and on-premises instances | Database Administration Lead |
| DATA-022 | Data Leakage Detection and Response | Implement automated detection of data leakage events across cloud storage, email, collaboration tools, code repositories, and dark web monitoring with defined incident response playbooks | Detective / Corrective | IR-4, SI-4, AU-6 | A.5.24, A.8.16 | 3.13, 17.4 | DSP-08, SEF-03 | DSS02.01, DSS02.02 | ISM-0123 | Continuous monitoring for data exfiltration attempts across all channels; assumes breach and monitors accordingly | Detection capabilities must span SaaS collaboration tools, cloud storage, CI/CD pipelines, and on-premises egress points | Security Operations Lead |
| DATA-023 | Data Segmentation and Isolation | Implement logical and physical data segmentation using separate storage accounts, encryption boundaries, access policies, and network controls to prevent unauthorized cross-tenant or cross-environment data access | Preventive | SC-4, SC-39, AC-4 | A.8.22, A.8.31 | 3.12 | DSP-13, IVS-06 | DSS05.02, DSS05.03 | ISM-1059 | Micro-segmentation at the data layer prevents lateral data access even within compromised environments | Segmentation must leverage cloud-native isolation constructs (resource groups, VPCs, encryption boundaries) consistently across providers | Security Architecture Lead |
| DATA-024 | Cloud Access Security Broker (CASB) Deployment | Deploy CASB capabilities for visibility, compliance enforcement, data security, and threat protection across all sanctioned and unsanctioned SaaS/IaaS/PaaS services | Detective / Preventive | AC-4, SI-4, SC-7 | A.8.12, A.8.23 | 3.13, 13.3 | DSP-08, STA-02 | DSS05.02, DSS05.07 | ISM-0263 | CASB acts as a policy enforcement point for data access aligned with Zero Trust proxy-based architectures | CASB must provide consistent policy enforcement for multi-cloud SaaS, IaaS APIs, and shadow IT discovery | Security Architecture Lead |
| DATA-025 | Secrets and Credentials in Data Protection | Prevent and detect storage of secrets, credentials, API keys, certificates, and tokens within data stores, code repositories, configuration files, and logs through automated scanning and vaulting | Preventive / Detective | IA-5, SC-12, SC-28 |
